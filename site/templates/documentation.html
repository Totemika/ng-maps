Directives

map

Creates an instance of the google.maps.Map class.

Property Type Description
center array Initial center of the map. Takes an array of two numbers, [lat, lng].
events object See documentation for events . Available events can be found on the Google Maps API.
options function See documentation for options . Available options can be found on the Google Maps API.
zoom number Initial map zoom level.

marker

Creates an instance of the google.maps.Marker class.

Property Type Description
decimals number Integer which defines how many decimal places the coordinates are rouded to when the marker is dragged.
events object See documentation for events . Available events can be found on the Google Maps API.
lat number Instead of 'position', lat and lng can be used to separetly define the markers position.
lng number Instead of 'position', lat and lng can be used to separetly define the markers position.
options function See documentation for options . Available options can be found on the Google Maps API.
position array Position of the marker. Takes an array of two numbers, [lat, lng].

points

Creates one or more instances of the google.maps.Marker class

Property Type Description
coords array Array of positions for each marker. Each position is an array of two numbers, [lat, lng].
decimals number Integer which defines how many decimal places the coordinates are rouded to when the markers are dragged.
events object See documentation for events . Available events can be found on the Google Maps API.
options function See documentation for options . Available options can be found on the Google Maps API.
properties array Array of equal length to coords. Each index should be an object describing some properties for the cooresponding object in coords at that index.
visible boolean If false, all objects will be hidden from the map.

polygons

Creates one or more instances of the google.maps.Polygon class. Also simulates multipolygon objects.

Property Type Description
coords array Array of polygon and/or multipolygon coordinates. The structure is [[[[lat, lng]]]]. The first array contains all of the polygons / multipolygons. Each index is another array of each polygon within those (a single polygon would have a length of one, a multipolygon would have a length greater than one). Each index is an array of coordinate pairs, defined as [lat, lng].
events object See documentation for events . Available events can be found on the Google Maps API.
opacity number The opacity for each object. Should be a number between 0 and 100.
options function See documentation for options . Available options can be found on the Google Maps API.
properties array Array of equal length to coords. Each index should be an object describing some properties for the cooresponding object in coords at that index.
visible boolean If false, all objects will be hidden from the map.

Events

Event objects consist of key value pairs where keys are the names of events and values are callback functions which handle that event. Which events are available depends on which directive you're using. The callback functions typically pass the event (typically a google.maps.MouseEvent, the object, and MapObjects as parameters.

$scope.points = {
  events: {
    click: function(e, p, MapObjects) {
      // some action here
    },
    drag: function(e, p, MapObjects) {
      // some action here
    }
  } 
}

In this case, functions care bound to the click and drag option. Because this events object will be used as a property in a points directive, these functions will be bound to every marker in that directive. The syntax is the same for every directive, regardless of how many objects are in them.

Options

Options are defined as functions instead of as objects. This allows you to define the directive's options as a function of your data. The function should always return an object that follows the Google Maps object specification for that class.

points, geopoints, polygons, geopolygons, polylines, circles, rectangles

For directives which create multiple map objects, the options function has four arguemnts: the individual object, the map, the index of that object, and MapObjects. In the following example, each geopolygon's opacity would be a function of its index.

$scope.geopolygons = {
  options: function(p, map, i, MapObjects) {
    var opacity = 100 / i;
    return {
      fillColor: 'red',
      fillOpacity: opacity
    }
  }
}

map, marker, overlay, infowindow

For directives which create a single object, the function passes no arguments. Simply return an object which specifies the appropriate options.

$scope.marker = {
  options: function() {
    return {
      draggable: true
    }
  }
}
